本文件区分三种状态：

- **待验证**：实现已存在，但尚无成功的独立执行记录；

## 已实现

### 发布止血与版本治理

- 删除会在普通 push/PR 中执行发布、重建旁路源码、强推发布分支并删除现有 Release/tag 的 `publish-v1.2.0.yml`。
- 发布改为仅由 `v*` tag 触发，并构建触发 tag 所指向的精确提交。
- tag 必须与 `app.__version__` 一致。
- 已存在的 Release 会导致任务失败，不再删除或覆盖。
- 仅发布 job 获得 `contents: write`；普通 CI 和 Windows 构建保持只读。
- 安装器版本由应用单一版本源注入，不再单独硬编码。

注意：工作流引用 `production-release` Environment。仓库管理员仍需在 GitHub 中为该 Environment 配置人工审批规则，否则“引用 Environment”本身不等同于已经启用审批。

### 纸张识别和 GUI 状态

- 删除严格模式下无条件接受 `minAreaRect` 的回退。
- 纸张候选增加面积、凸性、矩形度、内部亮度、外部背景对比和可选纸张比例残差检查。
- 严格模式要求自动识别置信度不低于明确阈值。
- 低置信度候选只能进入人工四角确认，不能作为已确认纸张继续处理。
- GUI 不再把原图静默复制成“已校正图”；未确认透视时预处理、识别和导出会被阻止。
- GUI 保存自动/手工角点、置信度、警告和旋转记录，供报告使用。

### 尺寸语义

- 报告明确区分 `pixel`、`paper_mm` 和 `model_mm`。
- 仅依据纸张外边界得到的比例标记为 `paper_mm`。
- 只有用户通过已知实际尺寸或图纸比例完成独立校准时才标记为 `model_mm`。
- GUI 和 CLI 均提示纸面毫米不等于原始工程设计尺寸。

### 几何正确性

- 在全部正交化和端点吸附完成后再次执行：
  - 非有限坐标过滤；
  - 零长度过滤；
  - 最小长度过滤；
  - 端点顺序规范化；
  - 最终重复实体合并；
  - 谱系和历史合并。
- 在交点处分割线段，构建端点图。
- 报告增加完全重复、近似重复、悬空端点、小间隙、连接分量、闭合分量、开放分量和未解析内部交叉点指标。

### 分辨率与粗线处理

- 预处理核、噪声阈值、最小线长、Hough 阈值、桥接、吸附、共线和去重距离根据完整校正图尺寸统一缩放。
- 独立几何工具在没有图像上下文时才退回使用线段坐标跨度估算尺度。
- 默认一像素细线去噪仍保持关闭，不会因高分辨率自动启用中值滤波。
- 粗印刷线候选可保守地移动到距离变换中心脊线，使同一笔画的双边检测结果能够在后续去重中合并。
- 该中心化仍是启发式，不代表已经恢复墙体或中心线语义。

### GUI/CLI 共用处理链与报告

- 新增 `PipelineService`，GUI 和 CLI 共用预处理、检线、几何清理、拓扑、分类和辅助识别流程。
- 新增统一 `ReportBuilder`，两端报告使用同一结构。
- 活动 GUI 将透视保护、导出、图层复核和圆形确认拆分为独立模块，不再由单个处理函数重复实现整条链路。
- 报告包含输入、角点、透视置信度、处理参数、图像质量、运行时间、尺度来源、坐标模式、谱系、拓扑和警告。

### 人工语义复核与圆形实体

- GUI 提供逐实体图层复核表，显示自动图层、置信度、来源和分类理由。
- 人工改层会保留自动分类理由，同时记录 `manual_layer_review` 和具体覆盖历史。
- 圆形候选必须同时满足保守置信度阈值和人工逐项勾选，才会导出真正的 DXF `CIRCLE`。
- 所有圆形默认不导出；未达到阈值的候选即使被选择也会被拒绝。
- 已确认圆形进入独立 `CIRCLE_CONFIRMED` 图层，报告和验证器记录导出数量。
- 圆弧、文字、尺寸和建筑符号仍仅为辅助候选。

### 验证与静态检查

- `validation/validate_dxf.py`：生成机器可读 DXF audit、LINE/CIRCLE 实体、图层、边界、重复、非法几何和拓扑证据。
- `validation/freecad_import_check.py`：在 FreeCAD/FreeCADCmd 环境中调用 `importDXF.open()`，记录 FreeCAD 版本、输入 SHA-256、导入对象和错误。
- 单元测试已增加纸张负例、最终重复检查、GUI 透视门槛、分辨率缩放、共享服务、统一报告、人工图层复核、圆形确认和拓扑夹具。
- CI 增加 Ruff、compileall 和首轮核心 mypy 检查；旧 GUI 模块暂未纳入严格 mypy，避免用大范围忽略项伪造全项目类型安全。

## 发布门槛

只有同时满足以下条件，才能创建新的正式 Release：

1. CI 和 Windows 构建全部成功；
2. DXF audit 错误为 0；
3. 非有限、零长度、非正半径和完全重复实体为 0；
4. 固定 FreeCAD 版本导入成功并保留 JSON 证据；
5. 至少一组真实照片与 ground-truth CAD 完成明确误差验收；
6. Release tag 与应用版本一致；
7. Release 不覆盖既有 tag 或资产；
8. 当前 Git 索引不包含构建目录和本机缓存；
9. GitHub `production-release` Environment 已实际配置人工审批保护。
